Certinel · Start-to-Finish Setup

Get Certinel up and running

A plain-English checklist. Fill in your details below and every step & command on this page updates to match — then just follow them top to bottom. No technical background needed.

Two kinds of steps. 🖥️ Web steps happen in the Certinel website (point & click). ⌨️ Terminal steps happen once, in a black command window on your server — copy the box exactly and paste it in. We fill in your details for you, so you can’t get them wrong.
Where to find this guide: open this file (setup-guide.html) in any browser to begin — even before Certinel is installed. Once it’s installed, the same guide is linked inside the app as Setup Guide in the top bar. Stuck on a field? Click the little ? next to it for help.

0What you need first

Gather these before you start. If you don’t have one, ask the person who gave you Certinel.

✅ A Linux server (RHEL or AlmaLinux 9) that you can sign into, with administrator (sudo) rights.
✅ The Certinel installer files (a folder, or an certinel-offline-*.tar.gz file for sealed/offline networks).
✅ Your license file — a short text blob you received after purchase (issued from your vendor’s License portal). Community edition needs no license — skip the license step.
✅ Your OpenBao/Vault details (address + an AppRole role_id and secret_id) from whoever runs your Vault.
✅ For CAC sign-in: your client-certificate CA bundle (e.g., the DoD root/intermediate chain) for the web server.
Don’t have a server yet? That part is a one-time job for an IT admin. Once it exists, everything below is copy-paste or point-and-click.

1Install Certinel ⌨️

Run this from inside the Certinel files folder. It asks a few simple questions and does the rest.

run on your server

Prefer no questions? This pre-answers everything from your details above:

B · Sealed / offline network

On a connected machine, build a self-contained bundle, copy it across, then run its installer.

on a connected machine
./make-offline-bundle.sh        # makes certinel-offline-<version>.tar.gz
on the offline server
tar xzf certinel-offline-*.tar.gz
cd certinel-offline-*
sudo bash install/online-install.sh   # the bundle carries everything it needs
When it finishes, Certinel is running at https://{{FQDN}}/csr/. Open that in a browser to continue.

2Create the first administrator

    
The very first person to sign in becomes the administrator. You turn this on for the initial setup, and it switches itself off automatically once you exist.


    
⌨️ Turn on first-admin mode. On the server, open the settings file and set the bootstrap line to 1:

    

    
run on your server


sudo sed -i 's/^CSR_BOOTSTRAP_FIRST_ADMIN=.*/CSR_BOOTSTRAP_FIRST_ADMIN=1/' /etc/certinel/certinel.env
sudo systemctl restart certinel-api

    

    
⌨️ Turn on first-admin mode. Add CSR_BOOTSTRAP_FIRST_ADMIN: "1" to the
      app service’s environment: in your Compose file, then docker compose up -d to apply.
      Remove it (and re-up) once you’ve registered.

    
⌨️ Turn on first-admin mode. Add it under extraEnv: in your values file, then re-run helm upgrade:
      
certinel-values.yaml


extraEnv:
  CSR_BOOTSTRAP_FIRST_ADMIN: "1"

      Remove it and upgrade again after you’ve registered.


    
🖥️ Make your account. Go to https://{{FQDN}}/csr/ → Register →
       enter your name, email, and a password. You’re now the admin.

    
🖥️ Sign in with your CAC. Go to https://{{FQDN}}/csr/ with your card inserted.
       Your identity is read from the card and the first one in becomes the admin.


    
Already created a normal account and need to promote it? On the server run
      sudo certinel-bootstrap-admin your-username.

    
Security tip: once you can see the Admin menu, set
      CSR_BOOTSTRAP_FIRST_ADMIN=0 again (same file as above) so nobody else can self-promote.

  


  
  

    
3Install your license (not needed)

    
You chose Community — it’s free and needs no license. Skip to the next step.

    

      
Your license unlocks your edition’s features. It’s verified right on your server — nothing phones home.

      
🖥️ In Certinel, open Admin → License, paste your license blob into the box, and click Save.

      
It should immediately show Commercial Edition with a green check. If it says
        “invalid,” you probably copied an extra space — paste it again cleanly.

      
Prefer a file instead of pasting? Put the blob in /etc/certinel/license.txt, add
        CSR_LICENSE_FILE=/etc/certinel/license.txt to /etc/certinel/certinel.env, and restart with
        sudo systemctl restart certinel-api.

    

  


  
  

    
4Set your organization identity

    
This fills in the “who” on every certificate (country, organization, units). Set it once.

    
🖥️ Open Admin → CSR Subject.

    
🖥️ Pick a profile from the list (e.g., U.S. Department of Defense, Army, Navy…). It pre-fills
      Country = US and Organization = U.S. Government, and offers DoD/command unit tags (USEUCOM, DISA, …).

    
🖥️ Fill in Organization (your organization), Country, State/Province, City,
      and any Organizational Units (e.g., IT, Security). Click Save.

    
There’s a live preview of the certificate “subject” line so you can see exactly what it’ll look like.

6Turn on automatic signing (OpenBao)

    
This lets Certinel issue certificates itself in the web app, instead of you uploading signed files by hand.


    
⌨️ Add your Vault details to the settings file on the server, then restart. (These live only on the
      server, never in the web app — that’s deliberate.)

    

    
run on your server — your details filled in


sudo tee -a /etc/certinel/certinel.env >/dev/null <<'EOF'
CSR_OPENBAO_ADDR={{OB_ADDR}}
CSR_OPENBAO_PKI_MOUNT={{OB_MOUNT}}
CSR_OPENBAO_ROLE={{OB_ROLE}}
CSR_OPENBAO_ROLE_ID={{OB_RID}}
CSR_OPENBAO_SECRET_ID={{OB_SID}}
CSR_CAP_OPENBAO=1
EOF
sudo systemctl restart certinel-api

    

    
✅ Already wired. Your Compose file from Step 1 carries these OpenBao settings as app
      environment variables — nothing to add here. Just confirm them in the next step.

    
✅ Already wired. The openbao: block in your values file from Step 1 puts the
      address/mount/role in the pod and the role_id/secret_id in a Secret — nothing to add here. Confirm in the next step.


    
🖥️ In Certinel, open Admin → Signing / CA, choose provider OpenBao PKI,
      confirm the address / mount / role, set it as the default, and click Save & Test.

    
Save & Test does a real test sign. A green “OK” means you’re fully wired — Certinel can
      now issue certificates on its own.

    
🖥️ Hands-off issuing (optional): on a certificate template, turn on Auto-sign so matching requests
      are issued instantly with no approval click.

    
Need the Vault side built (the PKI role + AppRole)? That’s a one-time job for your Vault admin:
      a policy allowing <mount>/sign/<role> (and, for per-user audit, auth/token/create),
      an AppRole, and its role_id/secret_id.

8Issue your first certificate 🎉

    
🖥️ Create requests → fill in the host name(s) you need a certificate for → submit.
      It appears as a pending request.

    
🖥️ Open the request and click Approve & sign. Certinel signs it through OpenBao and it flips to
      issued — download the certificate and chain right there.

    
🖥️ Send the request to your certificate authority, then come back and upload the signed certificate to finish it.

    
That’s the everyday loop your team will use. You’re done setting up!

Good to know — Commercial

    
Commercial covers one managed domain. If you try to sign for a second different domain, Certinel
      will stop and tell you. Need more? Ask your vendor for additional domains (or the Unlimited edition), then
      paste the new license in Admin → License — the new limit applies right away.

  

  

    
Good to know — Government

    
Government includes everything plus: CAC / smart-card sign-in, DoD/HIPAA/NSA consent banners,
      and public-sector certificate profiles (DoD, Army, Navy, Air Force, Space Force, Marines, federal civilian) with
      command unit tags. Set the banner and profile in Admin → Authentication and
      Admin → CSR Subject. No domain limit.

  


  
  

    
If something looks wrong

    
The website won’t load. On the server run sudo systemctl status certinel-api;
      if it’s stopped, sudo systemctl restart certinel-api.

    
“Save & Test” for signing fails. Re-check the five CSR_OPENBAO_* lines in
      /etc/certinel/certinel.env (Step 6) and that the server can reach the OpenBao address.

    
Can’t see the Admin menu. You’re not an admin yet — redo Step 2 (first-admin mode), or have an
      existing admin promote you.

    
License says invalid. Copy the blob again with no extra spaces or line breaks, and make sure it’s the
      one issued for this customer.


    
Plain-English glossary

    

      
CSR
“Certificate Signing Request” — a request asking a Certificate Authority to issue a certificate.

      
OpenBao / Vault
A secure system that holds the signing key and actually issues the certificates for you.

      
AppRole (role_id / secret_id)
A username/password pair Certinel uses to talk to OpenBao. Keep the secret_id private.

      
CAC / mTLS
Signing in with a smart card / client certificate instead of a password.

      
FQDN
Your server’s full web address, like certinel.example.com.
